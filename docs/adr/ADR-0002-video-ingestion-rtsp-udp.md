# ADR-0002: Приём видеопотока (RTSP/UDP + офлайн-реплей)

- Статус: Принято
- Дата: 2026-03-04
- Авторы: Максим Яковенко, Михаил Скрыпник, Иван Провков

## Контекст
Система должна уметь:
- работать в онлайн-режиме (поток с БПЛА через edge-узел),
- воспроизводимо прогонять миссии офлайн для сравнения конфигураций и подсчёта метрик пилота.

## Решение
Принять два режима источника кадров:
1) Офлайн для первичной отладки: сервис умеет читать заранее сохранённые последовательности кадров для воспроизводимых прогонов (реплей миссий).
2) Онлайн для MVP: edge-узел принимает поток и передаёт в сервис инференса по RTSP/UDP (H.264/H.265).

## Рассмотренные альтернативы
- HTTP push кадров (каждый кадр как запрос).
- gRPC streaming.
- Только файлы (только офлайн, без онлайн режима).

## Последствия
Плюсы:
- RTSP/UDP — стандартный путь для видеопотока и легко поддерживается ffmpeg.
- Офлайн-реплей даёт воспроизводимость и сравнение конфигов на одном наборе миссий.

Минусы:
- RTSP/UDP требует устойчивости декодирования и реконнекта при обрывах.
- Нужно явно описать поведение буфера (сколько секунд держим, что делаем при лаге).

## Дальнейшие шаги
- добавить health-check для потока (нет кадров N секунд → событие ошибки).
- логировать ошибки декодирования и частоту кадров для интерпретации метрик.
