# ADR-0005: Формат артефакта модели и стратегия рантайма

- Статус: Предложено
- Дата: 2026-03-04

## Контекст
В MVP важны быстрые итерации (перебор моделей/конфигов, анализ ошибок). В target/production важны производительность и переносимость на edge.

## Решение
В MVP:
- основной артефакт модели — PyTorch (.pt) для скорости экспериментов.

В target:
- предусмотреть экспорт в ONNX и/или OpenVINO (если упираемся в latency/FPS на edge),
- хранить версионированные артефакты (model + config alerting + manifest миссий).

## Рассмотренные альтернативы
- TorchScript как основной формат.
- Только ONNX/OpenVINO с самого начала.
- Только TensorRT (как привязка к NVIDIA).

## Последствия
Плюсы:
- MVP быстрый в разработке и проще дебажить.
- Есть путь оптимизации под слабое железо, если понадобится.

Минусы:
- Экспорт требует отдельного пайплайна валидации (сравнить метрики до/после экспорта).
